## Objective

Implement strict transactional logic for Sales Cancellations/Returns and create a Smart Restock assistant using data analytics.

## Context

Tech Stack: NestJS, PostgreSQL 18, Angular 21.2.0.

## Tasks

1. **Cancellations & Returns (Database & Backend):**
   - Write a strict PostgreSQL Stored Procedure `sp_cancelar_factura(p_id_venta)`.
   - The SP MUST: Change invoice status to 'CANCELADA', insert a reverse movement in the `kardex` ('ENTRADA POR DEVOLUCIÓN') to restore stock, and invalidate the corresponding record in the `pago` table (or create a credit note).
   - Expose this via a `POST /api/v1/venta/:id/cancelar` endpoint wrapped in a NestJS Transaction (QueryRunner) as an extra safety layer.

2. **Smart Restock Assistant (Analytics):**
   - Create a SQL View `vw_smart_restock` that calculates:
     - Sales velocity (units sold in the last 30 days).
     - Current stock.
     - Profit margin per product.
   - The view must sort products prioritizing high-rotation + high-margin items that are near depletion.
   - Create `GET /api/v1/inventario/smart-restock` to serve this view.

3. **Angular Frontend (`Operaciones`):**
   - Add a highly visible "Cancelar/Devolver" button in the Invoice Details view, protected by an confirmation dialog to prevent accidental clicks.
   - Create a `SmartRestockComponent` featuring a data-dense `mat-table` displaying the purchasing suggestions generated by the SQL view.
